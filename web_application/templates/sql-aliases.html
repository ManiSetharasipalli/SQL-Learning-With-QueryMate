SQL Aliases

SQL aliases are temporary names assigned to columns or tables for the purpose of clarity and brevity in SQL queries.

They are especially useful in complex queries involving multiple tables or columns with long names.

Additional points about SQL aliases:

  • Aliases can be used to make column or table names more readable.
  • They are commonly used when the column or table names are lengthy or when multiple tables are being joined.
  • Aliases are specified using the AS keyword.
  • They can be used in SELECT, FROM, and WHERE clauses of SQL queries.

Examples

Example 1: Alias for Column Name

Assign an alias to a column:

SELECT product_name AS name
FROM products; Query editor »

Try this example in Query editor

This example assigns an alias "name" to the "product_name" column in the 'products' table.

Example 2: Alias for Table Name

Assign an alias to a table:

SELECT p.product_name, c.category_name
FROM products AS p
INNER JOIN categories AS c
ON p.category_id = c.category_id; Query editor »

Try this example in Query editor

This example assigns aliases "p" and "c" to the 'products' and 'categories' tables, respectively, for brevity in the query.

Benefits of SQL Aliases

  • Improved readability: Aliases make SQL queries easier to understand, especially for complex queries involving multiple tables or columns.
  • Brevity: They help reduce the length of SQL queries, making them more concise and easier to manage.
  • Clarity: Aliases provide clarity by giving descriptive names to columns or tables, enhancing the overall understanding of the query.
