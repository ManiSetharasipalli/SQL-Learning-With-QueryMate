Subqueries in SQL

In SQL, a subquery is a query nested within another query. It allows you to perform operations using the result of another query.

Subqueries can be used in various parts of a SQL statement, such as SELECT, FROM, WHERE, HAVING, and EXISTS clauses.

Additional points about subqueries in SQL:

  • A subquery is a query nested within another query.
  • It can be used to retrieve data from tables based on conditions specified in the outer query.
  • Subqueries can return single or multiple values and can be used with comparison operators such as IN, NOT IN, EXISTS, etc.

Syntax

            SELECT column_name(s)
            FROM table_name
            WHERE column_name OPERATOR (SELECT column_name FROM table_name WHERE condition);

Examples

1. Subquery in WHERE Clause

Retrieve customers who have placed orders:

SELECT customer_name
FROM customers
WHERE customer_id IN (SELECT DISTINCT customer_id FROM orders); Query editor »

2. Subquery in SELECT Clause

Get the total number of orders for each customer:

SELECT customer_name,
(SELECT COUNT(*) FROM orders WHERE orders.customer_id = customers.customer_id) AS num_orders

Benefits of Subqueries

  • Modularity: Subqueries allow for breaking down complex queries into smaller, more manageable parts.
  • Flexibility: They offer flexibility by enabling dynamic filtering and conditional logic based on the result of inner queries.
  • Expressiveness: Subqueries enhance the expressiveness of SQL queries, allowing for more sophisticated data retrieval and manipulation.
