SQL Syntax

What is SQL Syntax?

SQL syntax is a unique set of rules and guidelines to be followed while writing SQL statements. This tutorial gives you a quick start with SQL by listing all the basic SQL Syntax.

All the SQL statements start with any of the keywords like SELECT, INSERT, UPDATE, DELETE, ALTER, DROP, CREATE, USE, SHOW and all the statements end with a semicolon (;).

Case Sensitivity in SQL

SQL statements are generally case-insensitive. This means `SELECT` and `Select` have the same meaning.

However, MySQL, a popular database management system, is case-sensitive for table names. So, ensure table names match their exact casing in the database.

SQL Statements

This tutorial introduces various SQL statements, commonly compatible with MySQL, Oracle, Postgres, and SQL Server databases.

Most statements require a semicolon (;) at the end, acting as a separator. This allows including multiple statements on a single line.

SQL Commands

Data Manipulation Language (DML):

  • SELECT: Retrieves data from a table.
  • UPDATE: Modifies existing data in a table.
  • DELETE: Removes rows from a table.
  • INSERT INTO: Inserts new rows of data into a table.

Data Definition Language (DDL):

  • CREATE DATABASE: Creates a new database.
  • ALTER DATABASE: Modifies a database.
  • CREATE TABLE: Creates a new table.
  • ALTER TABLE: Modifies a table.
  • DROP TABLE: Deletes a table.
  • CREATE INDEX: Creates an index (search key) to improve performance.
  • DROP INDEX: Deletes an index.

Data Control Language (DCL):

  • GRANT: Grants permissions to users or roles to access and manipulate database objects.
  • REVOKE: Revokes previously granted permissions.

Data Query Language (DQL):

  • SHOW: Retrieves information about the database itself, such as tables, users, and permissions.

Transaction Control Language (TCL):

  • COMMIT: Saves the changes made within a transaction permanently.
  • ROLLBACK: Undoes the changes made within a transaction, reverting to the state before the transaction began.
