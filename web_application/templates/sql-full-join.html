SQL FULL JOIN Operation

The FULL JOIN operation returns all rows from both tables, including rows where there is no match in the other table.

It combines the results of both LEFT JOIN and RIGHT JOIN.

Additional points about the FULL JOIN operation in SQL:

  • The FULL JOIN operation returns all rows from both tables, matching rows where available and filling in NULLs where no match is found.
  • It is useful for finding unmatched rows between two tables.
  • Aliases can be used for table names to make the SQL code more readable.

The FULL JOIN operation returns all rows from both tables.

Syntax

The basic syntax of the SQL FULL JOIN Operation is as follows:

                SELECT column_name(s)
                FROM table1
                FULL JOIN table2
                ON table1.column_name = table2.column_name;

Examples

Example 1: Retrieving Unmatched Rows from Two Tables

Retrieve all customers along with their orders, including unmatched rows:

SELECT c.customer_id, c.customer_name, o.order_id
FROM customers c
FULL JOIN orders o
ON c.customer_id = o.customer_id;
Example 2: Retrieving Unmatched Rows from Employees and Departments

Retrieve all employees along with their departments, including unmatched rows:

SELECT e.employee_id, e.employee_name, d.department_name
FROM employees e
FULL JOIN departments d
ON e.department_id = d.department_id;
